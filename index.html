Naming Conventions
& Folder Structure

Aprende a organizar y nombrar correctamente tus assets en UE5 desde el primer día, siguiendo estándares reales de producción.

📁 Folder Structure 🏷 Naming Conventions ⚡ Production Ready
01

¿Por qué Naming Conventions?

Las naming conventions son el lenguaje común de tu proyecto. Un proyecto que las sigue puede ser gestionado, buscado y mantenido con increíble facilidad —sin importar cuántas personas trabajen en él.

🔍
Búsqueda Instantánea
Escribir T_ en el Content Browser filtra únicamente las texturas. Sin convenciones, buscar un asset es una pesadilla.
🤝
Trabajo en Equipo
Todos los miembros del equipo identifican el tipo de asset a simple vista, sin necesidad de abrir el archivo.
🔄
Migración Sin Conflictos
Mover assets entre proyectos sin chocar con nombres duplicados o colisiones de rutas.
📐
Escalabilidad Real
Lo que funciona con 50 assets debe funcionar con 5,000. Las convenciones garantizan eso desde el inicio.
Prefix_ + BaseAssetName + _Variant + _Suffix
PREFIX
Identifica el tipo de asset. Acrónimo seguido de guión bajo. Ej: BP_, SM_
BASE ASSET NAME
Nombre corto y reconocible del asset o grupo. Ej: Hero, Rock, Door
VARIANT (opcional)
Diferencia versiones. Descriptivo o numérico. Ej: _Evil, _01
SUFFIX (opcional)
Info adicional del tipo. Principalmente en texturas. Ej: _D, _N, _R

Ejemplos de la fórmula en acción

BP_Hero
SK_Goblin_Warrior
SM_Rock_01
T_Rock_D
MI_Metal_Rusty
T_BrickWall_N
ABP_Hero
WBP_HUD_Main
💡
PascalCase siempre: Cada palabra del BaseAssetName comienza en mayúscula, sin espacios. Correcto: DesertEagle · Incorrecto: desert eagle, desert_eagle
02

Prefijos & Sufijos por Categoría

Cada tipo de asset tiene un prefijo asignado. Memoriza los más comunes — los usarás todos los días.

Tipo de Asset Prefijo Sufijo Ejemplo Descripción
▸ Core Assets — Los más usados
Blueprint BP_ — BP_Hero Lógica del juego, actores, componentes
Static Mesh SM_ — SM_Rock_01 Objetos 3D sin animación (props, ambiente)
Skeletal Mesh SK_ — SK_Hero Meshes con esqueleto para animación
Material M_ — M_Metal_Rusty Material base / master material
Material Instance MI_ — MI_Metal_Gold Variante de un material base (más eficiente)
Material Function MF_ — MF_BlendLayers Función reutilizable dentro de materiales
Texture T_ _D / _N / _R ... T_Rock_D Imágenes 2D para materiales. Sufijo indica canal
Widget Blueprint (UI) WBP_ — WBP_HUD_Main Interfaces de usuario, HUDs, menús
▸ Animación
Animation Blueprint ABP_ — ABP_Hero Controla la lógica de animación del personaje
Animation Sequence A_ — A_Hero_Run Clip de animación individual
Animation Montage AM_ — AM_Hero_Attack Combinación de animaciones para gameplay
Blend Space BS_ — BS_Hero_Locomotion Mezcla de animaciones por velocidad/dirección
Skeleton SKEL_ — SKEL_Hero Jerarquía de huesos del personaje
Aim Offset AO_ — AO_Hero_Rifle Offset de puntería para control de dirección
▸ Efectos Visuales (VFX)
Niagara System NS_ — NS_Fire_Large Sistema de partículas Niagara (UE5 recomendado)
Particle System (Cascade) PS_ — PS_Smoke_Trail Sistema de partículas legacy (UE4 / compatibilidad)
▸ Audio
Sound Wave A_ — A_Hero_Footstep Archivo de audio base importado
Sound Cue A_ _Cue A_Explosion_Cue Lógica de audio: randomización, atenuación
▸ Inteligencia Artificial
Behavior Tree BT_ — BT_Enemy_Patrol Árbol de comportamiento del NPC
Blackboard BB_ — BB_Enemy Memoria/datos del Behavior Tree
AI Controller AIC_ — AIC_Enemy Controlador de IA del personaje
▸ Blueprints Especiales & Datos
Blueprint Component BP_ Component BP_InventoryComponent Componente reutilizable para Blueprints
Blueprint Interface BPI_ — BPI_Interactable Interfaz para comunicación entre Blueprints
Data Table DT_ — DT_WeaponStats Tabla de datos tipo CSV para balance de juego
Enumeration E — EWeaponType Lista de opciones (sin guión bajo en el prefijo)

Sufijos de Texturas

Las texturas son el único tipo de asset que usa sufijos de forma extensa. El prefijo siempre es T_, pero el sufijo indica qué canal contiene la imagen.

_DDiffuse / Albedo / Base Color
_NNormal Map
_RRoughness Map
_MMetallic Map
_EEmissive Map
_AOAmbient Occlusion
_AAlpha / Opacity Mask
_SSpecular Map
_BBump Map
🎯
Texture Packing: Es común empaquetar múltiples canales en una sola textura para ahorrar memoria. Los sufijos se combinan.

Ejemplo: RGB con Emissive, Roughness y AO → T_Rock_ERO
Diffuse con Alpha en canal A → T_Leaf_DA
03

Estructura de Carpetas

La estructura de carpetas es igual de importante que las naming conventions. Violaciones en cualquiera de las dos genera caos innecesario en el proyecto.

📌
Principio fundamental: Organiza por feature / lógica de gameplay, no por tipo de asset. Carpetas como Meshes/, Textures/ o Materials/ son redundantes cuando usas prefijos correctamente — el Content Browser ya filtra por tipo.

Reglas de nomenclatura de carpetas

✓ Correcto
✅Usar PascalCase para nombres de carpetas
✅Todo dentro de Content/NombreProyecto/
✅Solo letras A-Z, números 0-9 y guión bajo
✅Nombres cortos, descriptivos y en inglés
✅Carpeta Maps/ dedicada para todos los levels
✅Carpeta Core/ para assets fundamentales del proyecto
✗ Incorrecto
❌Espacios en nombres: My Assets
❌Assets sueltos en la raíz de Content/
❌Caracteres especiales: @, #, !, -
❌Carpetas por tipo redundante: Textures/, Meshes/
❌Caracteres Unicode o acentos: Animación
❌Carpetas de trabajo personal mezcladas con el proyecto

Estructura recomendada — Visión general

Content/ ├── MyProject/ ← Carpeta raíz del proyecto (NUNCA assets fuera de aquí) │ ├── Art/ ← Assets de arte/ambiente (entorno, decoración) │ │ ├── Environment/ │ │ ├── Nature/ │ │ └── Industrial/ │ ├── Characters/ ← Un subfolder por personaje │ │ ├── Hero/ │ │ │ ├── Animations/ │ │ │ └── Audio/ │ │ ├── Enemy/ │ │ └── Common/ ← Assets compartidos entre personajes │ ├── Core/ ← ⚠️ Blueprints fundamentales del proyecto │ │ ├── Characters/ │ │ ├── GameModes/ │ │ └── Weapons/ │ ├── Effects/ ← VFX (Niagara, partículas) │ │ ├── Fire/ │ │ ├── Smoke/ │ │ └── Combat/ │ ├── Maps/ ← Todos los levels van aquí │ │ ├── Level_01/ │ │ └── Level_02/ │ ├── MaterialLibrary/ ← Master materials y funciones compartidas │ │ ├── Master/ │ │ ├── Instances/ │ │ └── Utility/ │ ├── Placeables/ ← Actors/props que se instancian en el mundo │ │ ├── Weapons/ │ │ └── Pickups/ │ ├── Sounds/ │ ├── UI/ │ └── Developers/ ← Sandbox personal (oculto por defecto) │ ├── ChristianMendez/ │ └── JuanGarcia/ └── ← Nunca assets aquí en la raíz de Content
⚠️
Core es zona restringida: La carpeta Core/ contiene los Blueprints fundamentales como GameMode, PlayerController, GameState. Solo desarrolladores senior deberían modificar estos assets. Los estudiantes deben hacer cambios en clases hijas, nunca en los Core assets directamente.
🧪
Developers folder: Usa Developers/TuNombre/ como área de sandbox personal para probar assets que aún no están listos. Estos assets no aparecen en búsquedas de otros miembros del equipo.

Reglas de Oro

Estos son los principios que debes interiorizar como ley. Sin excepciones.

⚖️
Las convenciones son ley
Un proyecto que las sigue se puede gestionar, buscar y mantener con increíble facilidad. Tratar las naming conventions como opcionales destruye la productividad del equipo.
🏠
Todo dentro del namespace
Ningún asset fuera de Content/NombreProyecto/. La raíz del Content Browser es para ThirdParty y Engine, jamás para tu proyecto.
🚫
Sin espacios, jamás
Los espacios en nombres de assets y carpetas rompen herramientas de ingeniería, scripts de batch y procesos de build. Usa PascalCase sin excepción.
📂
Maps en su carpeta
Todos los levels deben vivir dentro de Maps/. Esto simplifica el cooking, lighting builds y procesos de QA enormemente.
🔒
Respeta Core
La carpeta Core/ envía un mensaje claro: no tocar. Haz tus cambios en clases hijas, no en los assets base.
🧹
Consistencia total
Todo el proyecto debe parecer creado por una sola persona, sin importar cuántos contribuyeron. La consistencia reduce el tiempo de onboarding de nuevos miembros.
🔥
Nunca muevas assets después de haberlos referenciado. Mover assets genera redirectors en cadena que pueden romper referencias en todo el proyecto. Organiza bien desde el día uno — el costo de reorganizar después es enorme.
